# PRD: P2 交互重设计 — 发起批测 + 配置展示

> 状态：已确认
> 版本：v0.3 Phase P2
> 关联 backlog：UX-09（配置表单分组）、FT-03（批测用例选取）

---

## 1. 背景与问题

### 发起批测 Modal

当前 Modal 只允许选择 Agent 版本和并发数，配置信息（测试用例、Checklist、评判维度、模型配置）为纯只读摘要，用户无法定制本次批测的评测范围。

**目标**：在保持 Modal 简洁结构的前提下，让用户能按需调整本次批测的测试范围。

### 裁判配置查看态

当前无查看态，页面始终是完整编辑表单。信息密度不合理，用户无法快速浏览已配置内容。

**目标**：提供结构化的只读展示，点击"编辑"后切换到表单态。

### 模型配置查看态

当前无查看态，同样始终为编辑表单。且双栏横向排列对小屏不友好，两个模型共用一个"保存"按钮但实际无耦合。

**目标**：纵向堆叠 + 分别编辑（每个模型独立 view/edit 状态）。

## 2. 发起批测 Modal

### 2.1 交互结构

保持 Modal 形式，布局从上到下：

```
┌────────────────────────────────────────────────┐
│ 发起批测                                  [×]  │
├────────────────────────────────────────────────┤
│                                                │
│ * 选择 Agent 版本                              │
│   [ 选择要测试的 Agent 版本            ▾ ]     │
│                                                │
│   并发数                                       │
│   [ 3 ]                                        │
│                                                │
│   测试用例（5/5）                              │
│   [ 正常配合用户, 信息模糊用户, 急躁用户 ▾ ]   │
│                                                │
│   Checklist 检查项（10/10）                    │
│   [ Agent 正确介绍了..., Agent 成功收集... ▾ ] │
│                                                │
│   评判维度（3/3）                              │
│   [ 任务完成度, 逻辑连贯性, 沟通技巧      ▾ ]  │
│                                                │
│   通过阈值                                     │
│   [ 2.0 ]                                      │
│                                                │
│ ── 模型配置（只读）────────────────────────── │
│   对练模型  追一 / Mistral-Small-3.1-24B       │
│   裁判模型  追一 / Qwen3-235B-A22B-Instruct   │
│                                                │
│                       [ 取消 ]  [ 发起批测 ]   │
└────────────────────────────────────────────────┘
```

### 2.2 组件与行为

| 字段 | 组件 | 默认值 | 说明 |
|------|------|--------|------|
| Agent 版本 | `Select` | 空 | 必填，仅显示连接测试通过的版本 |
| 并发数 | `InputNumber` | 3 | 范围 1-20 |
| 测试用例 | `Select mode="multiple" maxTagCount="responsive"` | 全选 | label 显示 `(已选/总数)` |
| Checklist 检查项 | `Select mode="multiple" maxTagCount="responsive"` | 全选 | option 包含内容文本 + 级别标签（必须/应该） |
| 评判维度 | `Select mode="multiple" maxTagCount="responsive"` | 全选 | option 为维度名称 |
| 通过阈值 | `InputNumber` | 项目配置值 | 范围 1-3，步进 0.1 |
| 模型配置 | 纯文本 | — | 只读展示，改模型需去项目配置页 |

### 2.3 工作流

| 场景 | 操作 |
|------|------|
| 快速运行 | 选 Agent 版本 → 点「发起批测」 |
| 定制运行 | 选 Agent 版本 → 调整多选或阈值 → 点「发起批测」 |

### 2.4 数据流

1. Modal 打开时加载项目的 testCases、judgeConfig、modelConfig
2. 多选框默认全选（所有 ID）
3. 提交时，若全部选中则不传对应 ID 字段（后端默认全量），部分选中则传选中 ID 列表
4. 后端按选择过滤后存入 `config_snapshot`

### 3.2 组件选型

| 元素 | 组件 | 说明 |
|------|------|------|
| 通过阈值 | `Typography.Text` | label + bold 数值，一行展示 |
| Checklist | `Table size="small"` | 列：序号、内容、级别（Tag 红=必须/橙=应该），无分页 |
| 评判维度 | `Card size="small"` × N | 每个维度一个 Card，标题=维度名，描述在副标题，内嵌 3/2/1 分表格 |
| 编辑按钮 | `Button` | 右上角，点击切换到现有 Form 编辑态 |

### 3.3 状态切换

- 默认查看态
- 首次进入且无数据（checklist + eval_dimensions 均空）→ 自动进入编辑态
- `hasAutoEntered` ref 防止 React Query refetch 重入

### 4.2 组件选型

| 元素 | 组件 | 说明 |
|------|------|------|
| 外层 | 两个 `Card size="small"` 纵向排列 | gap 16px |
| Card extra | `Button` "编辑" | 每个 Card 独立 |
| 厂商/模型 | `Typography.Text` | label secondary + value strong |
| 参数行 | 自定义 flex 行 | Temperature + Max Tokens 同行 |
| System Prompt | `div` 浅灰背景 | `Typography.Paragraph ellipsis={{ rows: 4, expandable: true }}` |
| 未配置 | 占位文本 | `Typography.Text type="secondary"` "未配置" |

### 4.3 分别编辑行为

- 每个 Card 有独立的 `editing` state（`editingSparring` / `editingJudge`）
- 点击某个 Card 的"编辑" → 该 Card 内容替换为 Form 控件（Select 厂商/模型、InputNumber、TextArea）
- 底部显示"保存"+"取消"按钮
- 保存调用 `updateModelConfig` API（只传当前 Card 对应的字段）
- 取消后恢复查看态
- 首次无数据（对应模型的 provider 为空）→ 该 Card 自动进入编辑态

## 5. Agent 版本 Modal 分组

恢复 Divider 三段分组：
1. **基础信息**：版本名称、描述
2. **连接配置**：Endpoint、请求方法、响应格式、认证方式、认证令牌
3. **协议配置**：请求模板、响应解析路径、结束信号相关

## 6. 后端接口变更

### BatchTestCreate schema 新增字段

```python
class BatchTestCreate(BaseModel):
    agent_version_id: UUID
    concurrency: int = Field(default=3, ge=1, le=20)
    test_case_ids: list[UUID] | None = None          # 新增
    checklist_item_ids: list[UUID] | None = None      # 新增
    eval_dimension_ids: list[UUID] | None = None      # 新增
    pass_threshold: float | None = None               # 新增（覆盖项目默认值）
```

### validate_and_create 逻辑

- `test_case_ids` 非空时按 ID 过滤测试用例
- `checklist_item_ids` 非空时按 ID 过滤 checklist 存入 snapshot
- `eval_dimension_ids` 非空时按 ID 过滤 eval_dimensions 存入 snapshot
- `pass_threshold` 非空时覆盖 judge_config 的默认阈值存入 snapshot

### _load_context 调整

- 从 snapshot 读取已过滤的 checklist_items 和 eval_dimensions
- 从 snapshot 读取 pass_threshold（如有覆盖值）
